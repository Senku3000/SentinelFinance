The Agent Pipeline

A multi-agent system built on LangGraph. Each agent specializes in a part of the analysis — routing, research, calculation, and synthesis.

Router

Analyzes intent — does this need research, a calculation, or both? Routes to the right agents. Never skips to clarification; always gathers evidence first.

Researcher

Searches 3 sources: your uploaded documents via FAISS (RAG), the financial knowledge base, and the live web via Tavily for prices and market data.

Analyst

The LLM writes the formula, Python executes it exactly. EMI, SIP future value, tax liability, ROI — no guessing, no rounding errors.

Strategist

Combines research + calculations + your profile into a direct, opinionated recommendation. Uses your actual numbers, gives real opinions.

How RAG Retrieval Works

When you upload a document, it's chunked, embedded, and stored. When you ask a question, we find the most relevant chunks.

1

Document Chunking

Your uploaded file is split into overlapping chunks of ~1000 characters. Each chunk retains context from surrounding text. Excel rows become "Column: Value" text lines.

2

Vector Embedding

Each chunk is converted to a 384-dimensional vector using the MiniLM model. These vectors capture semantic meaning — "salary" and "income" end up close together in vector space.

3

FAISS Storage

Vectors are stored in a per-user FAISS index. Each user's documents are completely isolated — your data never mixes with anyone else's.

4

Similarity Search

When you ask a question, it's embedded into the same vector space. FAISS finds the closest matching chunks using cosine similarity — retrieving the most relevant parts of your documents.

Precise Calculations, Not Guesses

LLMs can't do math reliably. So the Analyst writes Python code, and a sandboxed REPL executes it with perfect precision.

1

User Asks

"What's the EMI for a 50 lakh home loan at 8.5% for 20 years?"

2

LLM Writes the Formula

The Analyst agent translates the question into correct Python code with the right financial formula. It knows EMI, SIP, CAGR, tax slabs, etc.

3

Sandboxed Execution

Code runs in a restricted Python environment — only math, numpy, and basic operations allowed. No file access, no network, no imports. Safe and precise.

4

Exact Result

The Strategist gets the precise number and presents it: "Your EMI would be ₹43,391/month — that's 28.9% of your income."

math_repl.py
# LLM generates this code:
P = 5000000 # ₹50 lakh
R = 8.5 / 100 / 12 # monthly rate
N = 20 * 12 # 240 months
result = (P * R * (1+R)**N)
/ ((1+R)**N - 1)
>>> ₹43,391.07 /month
